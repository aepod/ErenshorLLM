This directory contains personality profiles for all known SimPlayers in Erenshor. These profiles are designed to drive LLM-powered dialog generation, providing stable personality anchors regardless of each SimPlayer's randomly assigned class, level, and guild.

## How to Use These Profiles

### For LLM Dialog Generation

1. Look up the SimPlayer's name in the profile files
2. Load their personality traits, chat style, interests, and quirks
3. Check their current class (runtime value) to adjust combat-related dialog
4. Check their current guild (runtime value) to apply guild interaction rules from the guild profiles
5. Check base-game personality flags (TypesInAllCaps, etc.) -- these should be applied AFTER LLM generation
6. The LLM should generate dialog that:
   - Matches the personality traits (tone, vocabulary, interests)
   - Respects the chat style (formal vs casual, verbose vs terse)
   - Incorporates quirks naturally (not every line, but regularly)
   - Accounts for guild relationships (see guild profiles)
   - Accounts for relationship with the player (OpinionOfPlayer, KnowsPlayer, playerFriend)

### For Personality Flags

The base game applies these transformations AFTER dialog text is generated:
- TypesInAllCaps, TypesInAllLowers, LovesEmojis, TypoRate etc.
- These should be applied as post-processing on the LLM output
- The LLM should generate "clean" text that matches the personality, and then the game's PersonalizeString system handles formatting

### For Class-Specific Dialog

Since random SimPlayers can be any class, their personality profiles are class-agnostic. The LLM should:
1. Load the personality profile for stable traits
2. Check the current class for combat-relevant dialog
3. Blend both: a "studious" Paladin speaks differently than a "studious" Arcanist

### For Guild-Specific Dialog

1. Check if the SimPlayer is in the same guild as the player
2. Check if the SimPlayer is in Friends' Club (Rival flag)
3. Apply the appropriate guild interaction rules from the guild profiles
4. Guild membership should modify TONE, not core personality
